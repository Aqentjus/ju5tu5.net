Host multible websites on linux

September 19, 2023 · Justus

To better follow this you should first read this

  1. Ok… First make sure youre in the correct place:
cd /var/www/html

  1. in this example I’m going to host just two sites but you can host as many as you want… In this example i want to have one site in domain ju5tu5.net and another at test.ju5tu5.net

  2. So to host two sites make a folder for each different site you want to make like in the picture:

  3. then place your website files in each of these folder

  4. next we need to change the directory we are in:

cd ~

cd /etc/apache2/sites-available/

  1. when inside this folder make a copy of the existing 000-default.conf file so you have two files according to the picture or as many files as you wish to have different websites:

  2. Now lets modify the first by typing this command:

sudo nano 000-default.conf
  1. In nano paste this in the the file(change the ServerName to your domain name where you want the website to run, and change DocumentRoot to the location where your website files are located):
<VirtualHost *:80>
        # The ServerName directive sets the request scheme, hostname and port that
        # the server uses to identify itself. This is used when creating
        # redirection URLs. In the context of virtual hosts, the ServerName
        # specifies what hostname must appear in the request's Host: header to
        # match this virtual host. For the default virtual host (this file) this
        # value is not decisive as it is used as a last resort host regardless.
        # However, you must set it for any further virtual host explicitly.
        ServerName ju5tu5.net

        ServerAdmin webmaster@localhost
        DocumentRoot /var/www/html/ju5tu5.net

        # Available loglevels: trace8, ..., trace1, debug, info, notice, warn,
        # error, crit, alert, emerg.
        # It is also possible to configure the loglevel for particular
        # modules, e.g.
        #LogLevel info ssl:warn

        ErrorLog ${APACHE_LOG_DIR}/error.log
        CustomLog ${APACHE_LOG_DIR}/access.log combined

        # For most configuration files from conf-available/, which are
        # enabled or disabled at a global level, it is possible to
        # include a line for only one particular virtual host. For example the
        # following line enables the CGI configuration for this host only
        # after it has been globally disabled with "a2disconf".
        #Include conf-available/serve-cgi-bin.conf
</VirtualHost>

# vim: syntax=apache ts=4 sw=4 sts=4 sr noet
  1. Now save changes and do this for the same website also:
sudo nano 001-default.conf
  1. and then then (also remember to make the same changes here)
<VirtualHost *:80>
        # The ServerName directive sets the request scheme, hostname and port that
        # the server uses to identify itself. This is used when creating
        # redirection URLs. In the context of virtual hosts, the ServerName
        # specifies what hostname must appear in the request's Host: header to
        # match this virtual host. For the default virtual host (this file) this
        # value is not decisive as it is used as a last resort host regardless.
        # However, you must set it for any further virtual host explicitly.
        ServerName test.ju5tu5.net

        ServerAdmin webmaster@localhost
        DocumentRoot /var/www/html/test

        # Available loglevels: trace8, ..., trace1, debug, info, notice, warn,
        # error, crit, alert, emerg.
        # It is also possible to configure the loglevel for particular
        # modules, e.g.
        #LogLevel info ssl:warn

        ErrorLog ${APACHE_LOG_DIR}/error.log
        CustomLog ${APACHE_LOG_DIR}/access.log combined

        # For most configuration files from conf-available/, which are
        # enabled or disabled at a global level, it is possible to
        # include a line for only one particular virtual host. For example the
        # following line enables the CGI configuration for this host only
        # after it has been globally disabled with "a2disconf".
        #Include conf-available/serve-cgi-bin.conf
</VirtualHost>

# vim: syntax=apache ts=4 sw=4 sts=4 sr noet
  1. Now to apply these changes run these commands:
sudo a2ensite 000-default.conf

sudo a2ensite 001-default.conf

sudo systemctl restart apache2
  1. ok now you need to make sure that you have the domains set up correctly in your dns provider so both web addresses are pointing to your servers ip address. test.ju5tu5.net and ju5tu5.net in my case
